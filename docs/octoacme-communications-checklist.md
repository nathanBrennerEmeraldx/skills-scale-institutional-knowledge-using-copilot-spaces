# OctoAcme Communications Coordination Checklist

## Purpose
Guide Communications Coordinators in developing and executing effective communication strategies to ensure stakeholders receive timely, accurate project information and maintain alignment throughout the project lifecycle.

---

## Phase 1: Communication Planning (Project Kickoff)

### Stakeholder Analysis
- [ ] Identify all stakeholder groups (internal and external)
- [ ] Document stakeholder interests and information needs
- [ ] Assess stakeholder influence and impact on project
- [ ] Determine preferred communication channels for each group
- [ ] Create stakeholder contact list with roles and contact info

### Communication Strategy
- [ ] Define communication objectives for the project
- [ ] Establish communication frequency for each stakeholder group
- [ ] Select communication methods (email, meetings, dashboards, wiki)
- [ ] Determine escalation communication protocols
- [ ] Document communication plan in project charter

### Communication Channels & Tools
- [ ] Set up project communication channels (Slack, Teams, etc.)
- [ ] Create project documentation repository or wiki
- [ ] Establish email distribution lists
- [ ] Set up project dashboard or status page
- [ ] Configure notification preferences and access permissions

### Communication Templates
- [ ] Customize weekly status update template for project
- [ ] Prepare milestone announcement template
- [ ] Create risk and issue escalation template
- [ ] Develop release announcement template
- [ ] Prepare project closure summary template

---

## Phase 2: Ongoing Communication (Execution)

### Weekly Status Updates
- [ ] Collect status information from Project Manager
- [ ] Review progress, risks, and upcoming milestones
- [ ] Draft weekly status update using template
- [ ] Review content with Project Manager for accuracy
- [ ] Distribute to appropriate stakeholder groups
- [ ] Archive status updates in project documentation

### Status Update Content Checklist
- [ ] Project overview and current phase
- [ ] Progress this week (accomplishments)
- [ ] Progress next week (planned activities)
- [ ] Risks, blockers, and issues
- [ ] Decisions needed or escalations
- [ ] Key metrics or milestones

### Cross-Team Coordination
- [ ] Facilitate information sharing between teams
- [ ] Coordinate meeting schedules and agendas
- [ ] Distribute meeting notes and action items
- [ ] Track and follow up on cross-team dependencies
- [ ] Ensure consistent messaging across teams

### Documentation Management
- [ ] Maintain central project documentation repository
- [ ] Keep project wiki or knowledge base up to date
- [ ] Organize documents for easy discovery and access
- [ ] Version control important documents
- [ ] Archive outdated documentation

---

## Phase 3: Milestone & Event Communication

### Milestone Communication
- [ ] Identify upcoming project milestones
- [ ] Coordinate milestone messaging with Project Manager and Product Manager
- [ ] Draft milestone announcement
- [ ] Schedule communication timing for maximum impact
- [ ] Distribute announcement to stakeholders
- [ ] Celebrate achievements with the team

### Event Coordination
- [ ] Schedule and send invites for key meetings (kickoff, reviews, retrospectives)
- [ ] Prepare and distribute meeting agendas in advance
- [ ] Set up meeting spaces (physical or virtual)
- [ ] Collect and distribute meeting materials
- [ ] Send meeting notes and action items post-meeting

### Change Communication
- [ ] Identify scope, timeline, or resource changes
- [ ] Draft change notification with rationale and impact
- [ ] Review with Project Manager and Product Manager
- [ ] Communicate changes to affected stakeholders
- [ ] Update project documentation to reflect changes

---

## Phase 4: Release Communication

### Pre-Release Communication (T-2 weeks)
- [ ] Draft release announcement with Release Manager
- [ ] Identify audiences for release communication (internal, customers, partners)
- [ ] Customize messaging for each audience
- [ ] Coordinate review of customer-facing messaging
- [ ] Schedule release announcement timing

### Release Announcement Content
- [ ] Release overview and key features
- [ ] Benefits and value to users
- [ ] Migration steps or breaking changes (if applicable)
- [ ] Known issues and workarounds
- [ ] Support resources and documentation links

### Internal Release Communication
- [ ] Brief customer support team on changes and known issues
- [ ] Notify sales team of new features and positioning
- [ ] Update internal knowledge base and FAQs
- [ ] Provide demo or training materials if needed
- [ ] Prepare internal FAQ document

### External Release Communication
- [ ] Publish release notes on public channels
- [ ] Send customer email announcement (if applicable)
- [ ] Update product documentation and help content
- [ ] Coordinate social media or blog post (if applicable)
- [ ] Monitor customer feedback and questions

### Post-Release Communication
- [ ] Send deployment completion notification
- [ ] Share early adoption metrics or feedback
- [ ] Communicate any post-release issues or hotfixes
- [ ] Gather and share customer testimonials or success stories

---

## Phase 5: Risk & Issue Communication

### Risk Communication
- [ ] Review Risk Register weekly with Risk Owner
- [ ] Identify risks requiring stakeholder awareness
- [ ] Draft risk update with mitigation actions
- [ ] Coordinate communication timing with Project Manager
- [ ] Distribute to appropriate stakeholders based on risk severity

### Issue Escalation Communication
- [ ] Identify issues requiring escalation
- [ ] Draft escalation notice with issue details and impact
- [ ] Coordinate with Project Manager on escalation path
- [ ] Notify affected stakeholders promptly
- [ ] Provide regular updates until issue is resolved

### Crisis Communication
- [ ] Activate crisis communication protocol for critical incidents
- [ ] Notify key stakeholders immediately
- [ ] Provide regular updates on incident status and resolution
- [ ] Coordinate with technical teams on messaging
- [ ] Send post-incident summary and action plan

---

## Phase 6: Project Closure Communication

### Closure Preparation
- [ ] Collect project outcomes and metrics from Project Manager
- [ ] Gather success stories and testimonials
- [ ] Document lessons learned from retrospective
- [ ] Prepare project closure summary
- [ ] Review closure communication with Project Manager

### Project Closure Announcement
- [ ] Draft project closure announcement
- [ ] Highlight achievements and outcomes
- [ ] Thank team members and stakeholders
- [ ] Share lessons learned (as appropriate)
- [ ] Communicate next steps or follow-on projects

### Knowledge Transfer
- [ ] Ensure all project documentation is complete and accessible
- [ ] Archive project materials in appropriate repositories
- [ ] Update organizational knowledge base with learnings
- [ ] Conduct handoff meetings for ongoing support or maintenance
- [ ] Document tribal knowledge for future reference

## Communication Cadence

### Daily
- Monitor project communication channels
- Respond to information requests
- Update documentation as needed

### Weekly
- Publish weekly status update
- Coordinate with Project Manager on content
- Review communication effectiveness

### Monthly
- Prepare executive summary for sponsors
- Review and update communication plan
- Assess stakeholder satisfaction

### Milestone-Based
- Announce milestone completion
- Coordinate release communications
- Share retrospective insights

---

## Communication Best Practices

### Clarity & Consistency
- Use clear, jargon-free language appropriate for audience
- Maintain consistent format and structure
- Be concise while providing sufficient detail
- Use visual aids (charts, diagrams) when helpful

### Timeliness
- Communicate proactively, not reactively
- Share updates on a regular schedule
- Provide advance notice for changes or decisions
- Respond promptly to information requests

### Accuracy
- Verify facts and data before distributing
- Review communications with Project Manager
- Correct errors quickly and transparently
- Maintain single source of truth for project status

### Audience Awareness
- Tailor messaging to audience needs and expertise
- Consider stakeholder interests and concerns
- Use appropriate level of detail for each audience
- Provide relevant context and background

---

## Collaboration & Interaction Points

### With Project Manager
- Daily: Coordinate on status and emerging issues
- Weekly: Collaborate on status updates and stakeholder communications
- As-needed: Support escalation and change communications

### With Product Manager
- Weekly: Align on product messaging and feature descriptions
- Release: Coordinate feature announcements and value communication
- As-needed: Translate product vision for different audiences

### With Release Manager
- Pre-release: Plan release announcement timing and content
- Release day: Coordinate deployment notifications
- Post-release: Communicate status and any issues

### With Risk Owner
- Weekly: Review risks requiring stakeholder communication
- As-needed: Draft risk escalation notices
- Monthly: Include risk summary in executive updates

### With Executive Sponsor
- Monthly: Prepare executive status briefings
- As-needed: Draft strategic communications
- Major decisions: Support communication of decisions to organization

---

## Notes for Communications Coordinator
- Build relationships with stakeholders to understand their needs
- Keep communications positive while being honest about challenges
- Archive all communications for future reference
- Continuously seek feedback on communication effectiveness
- Adapt communication style and frequency based on stakeholder preferences
